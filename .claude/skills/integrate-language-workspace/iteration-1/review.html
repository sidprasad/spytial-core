Nothing about Haskell needs the advanced path.\n\n### 2.2 Attach (Subproblem 2)\n\n**Seam: a `SpytialSpec` type class, one instance per type, composing automatically.**\n\n```haskell\nclass SpytialSpec a where\n spec :: Proxy a -> [SpecFragment]\n spec _ = [] -- empty default\n```\n\nUsers opt in per type:\n\n```haskell\ninstance SpytialSpec Step where\n spec _ = [ orientation [\"Step.Mix\"] [Below]\n , group [\"Ingredient\"] \"ingredients\"\n ]\n\ninstance SpytialSpec Recipe where\n spec _ = [ align [\"steps\"] Vertical ]\n```\n\nWhen `diagram x` runs, it walks every type encountered during relationalization, collects each one's `spec`, and concatenates them into a single YAML document. This is the Haskell version of Caraspace's \"specs on `Vec` and `T` compose independently\" \u2014 the type-class dispatch *is* the composition mechanism.\n\n**Drafted signatures** for the four required spec fragments:\n\n```haskell\norientation :: [Selector] -> [Direction] -> SpecFragment\nalign :: [Selector] -> Axis -> SpecFragment\ngroup :: [Selector] -> Text -> SpecFragment\n\n-- non-negotiable escape hatch:\ndiagramWithRawSpec :: Relationalize a => a -> Text -> IO ()\n```\n\nThe escape hatch is non-negotiable because typed DSLs always miss directives \u2014 the YAML reference has things like `inferredEdge`, `tag`, `projection` with `orderBy` that you won't want to model in the typed layer on day one.\n\n**Composition rule.** If both `SpytialSpec [Step]` and `SpytialSpec Step` are defined, both fire \u2014 the container spec sees `[Step]` selectors, the element spec sees `Step` selectors, and they coexist in the merged YAML. No precedence rule needed; the YAML language already handles directive composition.\n\n### 2.3 Present (Subproblem 3)\n\n**Pattern: explicit call, file-backed, browser-launched.**\n\n```haskell\ndiagram :: (Relationalize a, SpytialSpec a) => a -> IO ()\ndiagram x = do\n let json = encodeJSON (relationalize x)\n yaml = renderSpec (collectSpecs x)\n path <- writeTempHtml json yaml\n openBrowser path\n```\n\n**Bundle source: CDN for the prototype.**\n\nThe generated HTML embeds the JSON + YAML inline as `
