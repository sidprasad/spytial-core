A specialized React component for constructing Pyret data instances with an improved programming-language-friendly interface.

Key Features

Programming-Language Optimized

  • Constructor Dropdowns - Select from predefined Pyret data types (List, Tree, RBTree, Option)
  • Auto ID Generation - IDs generated automatically with compact display
  • PyretExpression Support - Free-form expression input to avoid structured editor pitfalls
  • Real-time Preview - Live Pyret code generation as you build

Improved UX vs InstanceBuilder

Feature InstanceBuilder PyretInputController
Data Type Support Generic atoms/relations Pyret-specific constructors
ID Management Manual ID entry required Auto-generated with compact view
Constructor Selection Free-form text input Dropdown with predefined types
Expression Support Not available Built-in PyretExpression type
Code Generation Via reify() method only Live preview panel

Usage Example

import { PyretInputController } from 'cnd-core'; // Basic usage <PyretInputController onChange={(instance) => console.log(instance.reify())} /> // With configuration <PyretInputController config={{ compactDisplay: true, allowExpressions: true, autoGenerateIds: true, customTypes: [ { name: 'MyData', constructors: ['Variant1', 'Variant2'], fields: { 'Variant1': ['field1'], 'Variant2': ['field1', 'field2'] } } ] }} onChange={(instance) => handleDataChange(instance)} />

How to Use

  1. Select value type (Data Constructor, Primitive Value, or Free Expression)
  2. For constructors: Choose from dropdown (empty, link, Leaf, Node, etc.)
  3. For primitives: Select type (Number/String/Boolean) and enter value
  4. For expressions: Enter any valid Pyret code
  5. Click "Add Value" to create the value
  6. Set a root value using the star button (‚òÜ/‚òÖ)
  7. View generated Pyret code in real-time

Supported Data Types

  • List: empty, link(first, rest)
  • Tree: Leaf(value), Node(value, left, right)
  • RBTree: Leaf(value), Red(value, left, right), Black(value, left, right)
  • Option: none, some(value)
  • Custom: Add your own data types via config

How It Addresses HCI Pitfalls

The component carefully avoids common structured editor problems:

  • Escape Hatch: PyretExpression allows free-form input when needed
  • Familiar Patterns: Dropdown selection follows standard UI patterns
  • Progressive Disclosure: Compact mode hides complexity by default
  • Flexible Architecture: Can be extended with custom data types
