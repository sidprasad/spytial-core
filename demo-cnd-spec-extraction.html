What this demo shows:

This demo demonstrates the new CnD specification extraction functionality. When you evaluate Pyret expressions that have a _cndspec method, the system will automatically extract the CnD specification and compose it with existing specs.

How to test CnD spec extraction:

  1. Try evaluating a simple expression like: Alice:Person
  2. Add some relations: alice.friend=bob
  3. Create an object with a _cndspec method (simulated example):
// Example of what the system looks for: data RBNode: | Black(value, left, right) | Red(value, left, right) sharing: method _cndspec(self): "constraints:\n - orientation:\n selector: right\n directions:\n - right\n - below" end end // When you evaluate: Black(42, empty, empty) // The system will call ._cndspec() and extract the specification

Note: This demo shows the extraction infrastructure. In a full Pyret environment, the _cndspec methods would be defined as part of data definitions.
