Onboarding playbook

Local development
  1. Install dependencies: npm install.
  2. Build browser bundles: npm run build:browser.
  3. Open this page locally: npm run dev and navigate to /webcola-demo/webcola-demo.html.
  4. Use Load guided sample to validate your toolchain.
Contribute with confidence
  • Use the REPL mount to probe constraints before rendering.
  • Switch layout format (Standard/Grid) to demo different experiences.
  • Share a link to this gh-pages to help others reproduce your steps.
  • When ready, replace the sample Alloy XML with your dataset and iterate.

What this demo actually does

Here is the exact sequence that runs when you click Apply Layout. Use it as a checklist when swapping in other IEvaluator implementations.

  1. Parse Alloy XML → AlloyInstance.parseAlloyXML reads your XML and the first instance is wrapped in AlloyDataInstance to provide typed accessors.
  2. Select an evaluator → The demo constructs a ForgeEvaluator (an IEvaluator) and initializes it with the Alloy source so it can answer relation queries. Swap in your own IEvaluator here if you need alternative semantics.
  3. Compile layout spec → parseLayoutSpec converts the YAML from the CND interface into a LayoutSpec object, surfacing parse errors in the UI if anything is invalid.
  4. Create a LayoutInstance → The compiled spec and evaluator are combined into LayoutInstance, which is responsible for producing coordinates and alignment edges from your data.
  5. Generate a layout → layoutInstance.generateLayout consumes the AlloyDataInstance and optional projections, returning an InstanceLayout. Constraint conflicts or group overlaps are reported through the modal error helpers.
  6. Render with WebCola → The resulting InstanceLayout is passed into the <webcola-cnd-graph> custom element, which renders an SVG using WebCola. Changing the Layout Format dropdown re-renders with the same layout data.
  7. Inspect + iterate → The Evaluator REPL mount lets you probe relations from your evaluator, while the CND textarea reflects the active YAML. Edit either input, then rerun Apply Layout to verify the pipeline end-to-end.
