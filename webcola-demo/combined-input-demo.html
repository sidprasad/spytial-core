This demo showcases the new Combined Input Component that simplifies the complex setup from the Pyret REPL demo.

üéØ What This Solves

The original Pyret REPL demo required users to:

  • Understand and manage multiple components (PyretReplInterface, webcola-cnd-graph, CnDLayoutInterface)
  • Write complex sync logic between components
  • Handle state management across components
  • Set up event handling for real-time synchronization

The Combined Input Component eliminates all of this complexity!

Simple API Usage

Instead of complex setup, you can now use a simple API:

const dataInstance = new CndCore.PyretDataInstance(v);
const evaluationContext = { sourceData: dataInstance };
const evaluator = new CndCore.Evaluators.SGraphQueryEvaluator();
evaluator.initialize(evaluationContext);
const pyretREPLInternal = window.__internalRepl;
const projections = {};

// Simple setup - just one function call!
CndCore.mountCombinedInput('my-container', {
  cndSpec: 'nodes:\n - { id: node, type: atom }',
  dataInstance: dataInstance,
  pyretEvaluator: pyretREPLInternal,
  projections: projections
});

Usage Instructions

Once the component is loaded, you can:

1. Use the Pyret REPL

  • Alice:Person - Add a simple atom
  • alice.friend = bob - Add a relation (after creating bob)
  • edge("1", "knows", "2") - Create an edge
  • help - Show all available commands
  • clear - Clear current instance

2. Configure Layout Constraints

  • Switch between No-Code (visual) and Code (YAML) views
  • Add node and edge constraints
  • Apply constraints in real-time

3. Watch Real-time Updates

  • üîÑ REPL changes instantly update the visualization
  • üìê Layout constraint changes trigger automatic re-rendering
  • ‚ö° All components stay synchronized automatically
