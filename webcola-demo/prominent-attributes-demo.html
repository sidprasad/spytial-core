New Feature: Make specific attributes more prominent than node labels!
Use the prominent: true option in attribute directives to make attributes stand out.
Example Data Structure
Consider a Person-Name-Age relationship where we want the person's name to be more prominent than their type label.
{ "atoms": [ { "id": "John", "type": "Person", "label": "Person" }, { "id": "JohnDoe", "type": "String", "label": "John Doe" }, { "id": "30", "type": "Number", "label": "30" } ], "relations": [ { "id": "name_rel", "name": "name", "types": ["Person", "String"], "tuples": [{ "atoms": ["John", "JohnDoe"], "types": ["Person", "String"] }] }, { "id": "age_rel", "name": "age", "types": ["Person", "Number"], "tuples": [{ "atoms": ["John", "30"], "types": ["Person", "Number"] }] } ] }
Layout Specification - Before (Traditional)
directives: - attribute: field: 'name' - attribute: field: 'age'
Result: Both name and age attributes appear smaller than the main "Person" label.
Layout Specification - After (With Prominent Attributes)
directives: - attribute: field: 'name' prominent: true - attribute: field: 'age'
Result: The name attribute ("John Doe") now appears larger and bolder than the "Person" label, while age remains normal size.

Advanced Usage with Selectors
You can also combine prominent attributes with selectors for fine-grained control:
directives: # Make name prominent only for Person atoms - attribute: field: 'name' selector: 'Person' prominent: true # Make title prominent only for Company atoms - attribute: field: 'title' selector: 'Company' prominent: true # Regular attribute for all - attribute: field: 'id'
Implementation Details
How it works:
  • Prominent attributes are rendered with 1.3x the font size of the main label
  • Prominent attributes use bold font weight
  • Prominent attributes are rendered before regular attributes
  • Regular attributes remain 0.8x the size of the main label
  • Fully backward compatible - existing configurations work unchanged
