Spatial Explorer

<spytial-explorer> is a drop-in replacement for <webcola-cnd-graph> that integrates Data Navigator with Spytial's constraint-derived spatial model. The result: keyboard navigation and spatial queries that no existing tool can derive from pixels alone.

Key insight: The REPL replaces drag-to-explore as an interaction primitive. Instead of visually scanning a diagram to answer "what is to the right of this node?", a screen reader user can type must.rightOf(Node0) and get a provably correct answer — grounded in the constraint graph, not pixel coordinates.

How to Use This Demo

Keyboard Navigation (Data Navigator)

  • Click "Enter diagram navigation" (or Tab to it) to start
  • ← → ↑ ↓ — move between nodes (behavior depends on navigation mode)
  • Navigation modes (press 1/2/3 to switch):
    • 1 Spatial — nearest neighbor in current layout
    • 2 Must-only — only constraint-guaranteed relationships (true in ALL valid layouts)
    • 3 Relations — follow named data relations (left, right, val)
  • Enter — drill into a group (navigate among group members)
  • Esc — leave group, or exit diagram navigation
  • The Spatial Context panel updates live with modality badges (MUST / CAN)

Spatial Queries (REPL)

  • must.rightOf(Node0) — nodes that must be right of Node0 in all valid layouts
  • can.leftOf(Node1) — nodes that can be left of Node1 in some valid layout
  • cannot.above(Node2) — nodes that cannot be above Node2 in any valid layout
  • Results are clickable — click a result atom to navigate to that node

Data Queries (REPL)

  • Node — all atoms of type Node
  • Node.left — follow the left relation from all Nodes
  • Node.left.val — chain relations: left children's values

Why This Matters

Traditional diagram accessibility tools (alt text, ARIA trees) describe what is in a diagram. Spytial answers why: the spatial layout is derived from constraints, not arbitrary positioning. This means:

  • must queries give guarantees — true regardless of layout variation
  • can/cannot queries reveal structural degrees of freedom
  • Data Navigator gets spatial semantics it can't derive on its own from the constraint graph

Using <spytial-explorer> in Your Own Page

 
<!-- 1. Include dependencies -->
<script src="d3.v4.min.js"></script>
<script src="cola.js"></script>
<script src="spytial-core-complete.global.js"></script>

<!-- 2. Drop in the element (same attributes as webcola-cnd-graph) -->
<spytial-explorer id="explorer" width="800" height="500"></spytial-explorer>

<script>
  // 3. Run the standard layout pipeline
  const instance  = new spytialcore.JSONDataInstance(jsonText);
  const evaluator = new spytialcore.SGraphQueryEvaluator();
  evaluator.initialize({ sourceData: instance });
  const layoutSpec = spytialcore.parseLayoutSpec(cndYaml);
  const li = new spytialcore.LayoutInstance(layoutSpec, evaluator, 0, true);
  const result = li.generateLayout(instance);

  // 4. Render (inherited from WebColaCnDGraph)
  await explorer.renderLayout(result.layout);

  // 5. One call enables all a11y features:
  //    Data Navigator overlay, spatial context, spatial + datum REPLs
  explorer.enableAccessibility(result.layout, result.validator, evaluator);
</script>
